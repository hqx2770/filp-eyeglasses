# 翻转拍训练系统 - 需求文档

---

## 1. 项目概述

### 1.1 项目背景

翻转拍（Flipper Lens）训练是儿童近视防控中常用的调节力训练方式。目前市面上存在部分 Web 端训练系统，但功能分散、体验不一。本项目旨在设计并实现一套**基于 HTML5 的翻转拍训练系统**，以 Web 页面形式提供标准化、可配置的视觉训练功能，替代传统纸质训练卡片，实现数字化、数据化管理。

### 1.2 产品定位

- **产品形态**：纯 Web 端应用（HTML5 + JavaScript），无需安装
- **目标用户**：视光机构 / 眼科医院 / 家庭（家长辅助儿童训练）
- **运行环境**：PC 浏览器、平板浏览器

### 1.3 竞品参考

参考现有 Web 训练系统（如 demo 目录中的截图所示），核心界面包含：
- 5×5 E 字视标网格（每个格子一个 E 字，红色方框标记目标）
- 计时器与得分显示
- 训练参数配置面板（镜片度数、E 字大小、颜色等）
- 控制按钮（退出、暂停、重做、看不见等）

## 2. 功能需求

### 2.1 核心训练模块

#### 2.1.1 训练视标生成

| 需求项 | 描述 |
|--------|------|
| 视标类型 | E 字视标（不同方向） |
| 视标排列 | 5×5 网格（25 个格子），每个格子都显示一个 E 字视标 |
| E 字方向 | 每个 E 字随机生成（上、下、左、右四个方向之一），每次回答后全部刷新 |
| 红色方框 | 标记当前目标格子，用户需要识别该格子 E 字的开口方向 |
| 交互方式 | 用户用方向键回答目标格子中 E 字的方向（↑=上 ↓=下 ←=左 →=右），回答后自动刷新新目标 |

#### 2.1.2 翻转拍模拟

| 需求项 | 描述 |
|--------|------|
| 正镜模式 | 模拟正镜片（+），训练放松调节 |
| 负镜模式 | 模拟负镜片（-），训练加强调节 |
| 交替翻转 | 正镜与负镜交替展示，用户需在不同模式下识别视标 |
| 镜片度数选择 | 支持 ±1.00、±1.50、±2.00、±2.50D 等常见度数 |

#### 2.1.3 交互流程

1. 系统生成 5×5 网格，每个格子随机显示一个方向的 E 字
2. 红色方框标记一个目标格子（默认中心位置）
3. 用户通过**键盘方向键**回答目标格子 E 字的开口方向（↑上 ↓下 ←左 →右）
4. 系统判断正误：正确 +5 分，错误不计分，记录正确/错误次数
5. 回答后自动刷新网格（所有 E 字方向重新随机）并选择新的目标格子
6. 正镜与负镜模式按设定时间自动交替（如每 30 秒翻转一次）
7. 训练达到设定时间后自动结束

### 2.2 用户控制功能

| 功能 | 描述 | 快捷键 |
|------|------|--------|
| 回答方向 | 回答目标格子 E 字的开口方向 | `←↑→↓` |
| 方向软键盘 | 平板/触屏设备可点击屏幕右下角上下左右方向键作答 | 触控 |
| 暂停 | 暂停计时和训练 | `Space` |
| 重做 | 重新生成当前轮次 | `R` |
| 看不见 | 标记当前视标无法看清，跳过本轮 | `C` |
| 退出 | 退出训练 | `Esc` |

### 2.3 参数配置面板

| 配置项 | 可选值 | 默认值 |
|--------|--------|--------|
| 镜片度数 | ±1.00 / ±1.50 / ±2.00 / ±2.50D | ±2.00D |
| E 字大小 | 小 / 中 / 大 | 中 |
| 显示缩放 | 40% / 50% / 60% / 75% / 100% | 50% |
| 翻转间隔 | 10s / 20s / 30s / 45s | 30s |
| 训练时长 | 1min / 3min / 5min / 10min | 3min |
| 视标颜色 | 黑底白字 / 白底黑字 / 红绿对比 | 白底黑字 |

### 2.4 数据记录与报告

#### 2.4.1 实时统计

- 当前得分（每次正确 +5 分）
- 正确次数 / 错误次数
- 当前镜片模式（正镜 / 负镜）
- 剩余时间

#### 2.4.2 训练报告

训练结束后生成报告，包含：
- 总训练时长
- 总翻转次数（正镜→负镜交替次数）
- 正确次数、错误次数、正确率
- 正镜模式正确率 & 负镜模式正确率
- 训练结果评级（优秀 / 良好 / 合格 / 需加强）

#### 2.4.3 历史记录

- 支持查看历史训练记录列表
- 可筛选日期、镜片度数
- 趋势图表展示（正确率变化曲线）

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 首屏加载时间 | ≤ 2 秒 |
| 视标切换延迟 | ≤ 50ms（确保训练节奏）|
| 响应式设计 | 适配 1920×1080、1366×768、平板竖屏 |

### 3.2 用户体验要求

- 大字号显示，便于远距离观看
- 平板浏览器默认按 50% 显示缩放启动，后续可在参数面板继续调大或调小
- 高对比度配色（符合视光训练标准）
- 简洁界面，减少视觉干扰
- 支持全屏模式（F11）

### 3.3 兼容性

- Chrome 80+、Edge 80+、Firefox 78+
- 无需后端服务，纯前端实现（本地存储历史记录）

## 5. 技术实现方案

### 5.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 结构 | HTML5 | 语义化标签 |
| 样式 | CSS3 | Flexbox/Grid 布局，CSS 变量主题 |
| 交互 | Vanilla JavaScript | 无需框架，轻量高效 |
| 存储 | LocalStorage | 本地持久化训练记录 |
| 图表 | 可选 Canvas/SVG | 训练趋势图 |
| 构建 | 无构建工具 | 直接部署静态文件 |

### 5.2 模块划分

```

├── index.html          # 主入口页面
├── css/
│   └── style.css       # 样式
├── js/
│   └── app.js          # 全部逻辑（config/stimulus/timer/flipper/scorer/storage/keyboard/app）

```

### 5.3 核心算法

#### 5.3.1 网格生成与方向随机化

```
网格: 5×5（25 个格子），每个格子一个 E 字
E 字方向池: [上, 下, 左, 右]
grid[row][col] = 随机方向
目标格子: 随机选择 (targetRow, targetCol)，默认从中心 (2,2) 开始
每次回答后: 全部 E 字方向重新随机，目标格子重新随机
```

#### 5.3.2 翻转拍交替逻辑

```
状态机:
  IDLE -> 等待开始
  TRAINING -> 训练中（正镜->等待->负镜->等待->正镜...）
  PAUSED -> 暂停中
  FINISHED -> 训练结束

翻转流程:
  1. 显示当前镜片模式（正镜 +2.00D）
  2. 等待用户回答目标 E 字方向
  3. 到达翻转间隔时间后，切换至负镜 -2.00D
  4. 显示提示"请翻转镜片"，短暂延迟后刷新网格
  5. 循环直至训练时间结束
```

#### 5.3.3 计分规则

```
单次正确: +5 分（固定）
错误: 不计分
总正确率 = 正确次数 / (正确次数 + 错误次数) * 100%

特殊:
  "看不见" -> 不计入总数
```

## 7. 验收标准

| 序号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 训练流畅性 | 视标切换无明显延迟，帧率稳定 |
| 2 | 翻转逻辑 | 正负镜交替准确，翻转间隔可配置 |
| 3 | 计分准确 | 每次正确 +5 分，正确/错误次数统计无误 |
| 4 | 网格交互 | 5×5 网格每格显示 E 字，方向键回答方向判断准确 |
| 5 | 数据持久化 | 刷新页面后历史记录不丢失 |
| 6 | 报告完整 | 训练报告包含所有要求指标 |
| 7 | 浏览器兼容 | Chrome/Edge/Firefox 功能一致 |
